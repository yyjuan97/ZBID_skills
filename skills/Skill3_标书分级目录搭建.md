## 一、核心原则

**目录结构=评分结构+技术响应逐条覆盖**

目录必须同时满足两个要求：
1. 章节结构对齐评分项，评审翻目录能对上评分表；
2. 每条技术响应表中的技术要求都有独立标题，确保逐条响应不遗漏。

**目录范围限定**：仅包含技术方案相关章节，不出现商务、价格、资质、财务等非技术方案章节。

---

## 二、目录层级体系

层级不限三级，允许多层嵌套，直到每条技术要求都有独立标题为止。

| 层级 | 映射对象 | 说明 |
|------|---------|------|
| 第1层 = 章 | 评分大类/技术领域 | 评分表中的评分维度，或技术领域大类 |
| 第2层 = 节 | 评分项 | 评分表中的每个评分项，名称体现评分项关键词 |
| 第3层 = 小节 | 评分点/技术子领域 | 评分项下的得分点，或技术领域的子分类 |
| 第4层 = 条 | 技术要求 | 技术响应表中每条具体技术要求（T-001、T-002……） |
| 第5层及以上 | 技术要求的细化维度 | 仅当一条技术要求需要拆分多个方面详细设计时使用 |

**关键规则：技术响应表每条技术要求必须有独立标题位置，不得多条合并为一个标题。**

## 三、目录构建步骤

### 3.1 步骤一：从评分对标表提取章节骨架

逐行读取{项目}/01_解析与基准/评分对标表.md，按以下规则映射：

| 评分对标表中的情况 | 目录处理 |
|-------------------|---------|
| 评分项本身就是一个完整的技术领域 | → 升为第1层（章） |
| 评分项是某个技术领域下的子项 | → 归入对应第1层下，作为第2层（节） |
| 评分项包含多个独立得分点 | → 第2层（节）下拆出多个第3层（小节） |
| 评分项是简单的描述性要求 | → 作为第2层（节），是否设子章节取决于有无对应技术要求 |

**章节名称必须体现评分项关键词**：

| 错误写法 | 正确写法 | 原因 |
|---------|---------|------|
| "网络方案" | "网络架构设计与设备选型" | 对应评分项"网络架构及设备选型" |
| "系统安全" | "安全体系设计与防护方案" | 对应评分项"安全防护方案" |
| "服务方案" | "售后服务与运维保障方案" | 对应评分项"售后服务及运维" |

### 3.2 步骤二：从技术响应表逐条补充子章节（⚠️核心步骤）

**每个评分项对应的章节下，必须将技术响应表中归属于该评分项的技术要求逐条展开为独立标题。**

操作方法：
1. 读取{项目}/01_解析与基准/技术响应表.md全部行；
2. 根据每行技术要求的归属（属于哪个评分项/技术领域），分配到对应的章节下；
3. 每行技术要求生成一个独立标题，标题名=该技术要求的核心内容概括；
4. ★号参数必须设独立标题，不得合并。

| 技术响应表中的情况 | 目录处理 |
|-------------------|---------|
| 单条技术要求（如T-001：核心交换机背板带宽≥2Tbps） | → 第4层独立标题"核心交换机选型与配置" |
| 同一设备/系统下的多条技术要求 | → 先设第3层"XX设备/系统详细设计"，其下每条技术要求设第4层独立标题 |
| ★号技术要求 | → 必须有独立标题，标题名体现参数核心内容 |
| 定性技术要求（如"提供完善的运维管理体系"） | → 独立标题"运维管理体系设计" |
| 设备清单/配置表类要求 | → 独立标题"设备配置清单"，正文中用表格逐项响应 |

**构建示例**（假设评分项"网络架构及设备选型"下有8条技术要求）：

```
第3章 网络架构设计与设备选型                 ← 第1层：评分项
  3.1 网络总体架构设计                       ← 第2层：总体设计
  3.2 核心层网络详细设计                     ← 第3层：技术子领域
    3.2.1 核心交换机选型与配置               ← 第4层：T-001技术要求
    3.2.2 核心交换机背板带宽响应              ← 第4层：T-002 ★号技术要求
    3.2.3 核心交换机转发性能响应              ← 第4层：T-003技术要求
  3.3 汇聚层与接入层详细设计                 ← 第3层：技术子领域
    3.3.1 汇聚交换机选型与配置               ← 第4层：T-004技术要求
    3.3.2 接入交换机选型与配置               ← 第4层：T-005技术要求
  3.4 网络安全设备详细设计                   ← 第3层：技术子领域
    3.4.1 防火墙部署方案                    ← 第4层：T-006 ★号技术要求
    3.4.2 入侵检测系统配置                  ← 第4层：T-007技术要求
  3.5 网络管理与监控                         ← 第3层：技术子领域
    3.5.1 网管平台部署与配置                ← 第4层：T-008技术要求
```

### 3.3 步骤三：废标项覆盖确认

| 情况 | 处理方式 |
|------|---------|
| 废标条款涉及技术方案内容 | 归入对应技术章节中，不单独设"废标响应"章节 |
| 废标条款属于资格/资质要求 | 不在技术方案目录中出现 |
| 废标条款属于★号实质性技术参数 | 必须有独立标题，确保逐项响应 |

### 3.4 步骤四：招标文件章节结构适配

| 情况 | 处理方式 |
|------|---------|
| 招标文件明确列出技术方案章节名称 | 优先遵循，章节名称不改动 |
| 招标文件要求章节名与评分项不对应 | 以招标文件章节名为准，子章节对齐评分项 |

## 四、章节排序规则

### 4.1 排序优先级（从高到低）

1. **招标文件明确要求前置的章节**；
2. **涉及★号/实质性废标条款的章节**；
3. **高分值评分项对应章节**（按{项目}/01_解析与基准/评分对标表.md分值从高到低）；
4. **低分值评分项对应章节**；
5. **无直接评分但技术方案需要的支撑章节**（如项目概况）。

### 4.2 同优先级内排序

- 同为高分章节：按技术逻辑先后排序（先总体后局部、先设计后实施、先建设后运维）；
- 同为低分章节：按评分表中出现顺序排列。

### 4.3 子章节内部排序

同一章节下的技术要求子章节排序：
- ★号/实质性参数优先排布；
- 按技术响应表中出现的顺序排列；
- 同一设备/系统下的参数按参数重要性排列（核心参数在前）。

## 五、章节命名规则

### 5.1 各层级命名原则

| 层级 | 命名原则 | 示例 |
|------|---------|------|
| 第1层 | 体现评分大类/技术领域 | "网络架构设计与设备选型""安全体系设计与防护方案" |
| 第2层 | 体现评分项关键词 | 评审看名称能对应评分表 |
| 第3层 | 体现技术子领域分类 | "核心层网络详细设计""汇聚层与接入层详细设计" |
| 第4层 | 体现具体技术要求的核心内容 | "核心交换机选型与配置""防火墙部署方案" |
| 第5层+ | 技术要求的细化维度 | 仅当一条技术要求需拆分多方面时使用 |

### 5.2 技术要求子标题命名方法

标题名从技术响应表中提炼：
- 有明确参数的技术要求：标题名=设备/系统名+参数类别，如"核心交换机背板带宽响应"；
- 定性技术要求：标题名=要求概括，如"运维管理体系设计"；
- ★号技术要求：标题名必须明确体现★号参数内容，但不标注★号本身。

### 5.3 命名禁止

| 禁止 | 正确做法 |
|------|---------|
| 标题中写评分分值 | 分值录入风控台账 |
| 标题中写"★号条款"等标注 | 废标信息录入风控台账 |
| 章节名称过于笼统 | 名称必须包含评分项/技术要求关键词 |
| 出现商务/价格/资质类章节 | 删除此类章节 |
| 多条技术要求合并为一个标题 | 每条技术要求独立标题 |
| 招标文件有明确章节名却自行改名 | 优先用招标文件原名 |

## 六、目录排除规则

以下章节**不得出现在技术方案目录中**：

| 排除类型 | 识别特征 |
|---------|---------|
| 商务评分章节 | 含"企业资质""业绩""财务""信誉"等关键词 |
| 价格评分章节 | 含"报价""价格分""成本"等关键词 |
| 资格审查章节 | 含"资格证明""准入""许可"等（非技术参数部分） |
| 投标函/承诺书 | 含"投标函""投标保证金""承诺书"等 |

---

## 七、目录完整性校验

目录生成后，必须执行以下校验（校验结果不写入目录文件，仅在输出时向用户提示）：

| 校验项 | 校验方法 | 不通过处理 |
|--------|---------|-----------|
| 技术评分项100%覆盖 | 逐条对照评分对标表技术评分行，每项是否在目录中有对应章节 | 缺失评分项必须补充 |
| 技术要求100%覆盖 | 逐条对照技术响应表每行（T-001、T-002……），是否每行都有独立标题位置 | 缺失技术要求必须补充独立标题 |
| ★号参数100%覆盖 | 技术响应表中★号行是否每个都有独立标题 | ★号必须100%有独立位置 |
| 废标技术条款覆盖 | 废标核查表中技术相关废标条款是否有对应章节 | 缺失废标条款补充 |
| 无非技术章节 | 检查是否混入商务/价格/资质章节 | 立即删除 |

【禁止行为】
- 未审核目录直接生成正文内容
- 目录中出现商务/价格/资质等非技术方案章节

- 漏掉任何一个技术评分项
- 多条技术要求合并为一个标题（每条必须独立）
- ★号技术要求没有独立标题位置
