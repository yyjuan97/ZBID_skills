# Skill1｜招标文件智能解析拆分

【触发口令】调用技能1：招标文件智能解析拆分

【执行前置】用户提供招标文件（文件形式），无前置已完成数据。同时提供项目名称，AI据此在 projects/ 目录下创建项目文件夹结构。

【执行前必须读取】用户提供的招标文件，以及本文件（投标编制规范.md中的角色定位、流转规则、权限红线）

【执行行为指引】
1. 向用户说明当前步骤："正在执行步骤1：招标文件智能解析拆分，将清洗招标文件并按三分法（技术评分细则、技术需求规范、废标否决条款）拆分归类"；
2. 按本Skill规则执行解析，产出文件至项目文件夹；
3. 执行完毕后向用户输出以下信息：
   - 文件已生成：`{项目}/01_解析与基准/招标文件解析.md`
   - 解析概要：技术评分细则区X条、技术需求规范区X条、废标否决条款区X条
   - 提示用户："请审阅文件后回复【确认通过】进入步骤2，或【驳回重做：XX问题】重新解析"
4. 收到【确认通过】后，提醒用户下一步操作："请输入【调用技能2：生成三份基准文档】"
5. 收到【驳回重做】后，依据驳回说明重新解析，不得沿用旧版内容。

## 一、原文清洗规则

### 1.1 必须剔除的内容

以下内容与投标技术方案无关，识别后直接剔除：

| 剔除类别 | 识别特征 | 示例 |
|---------|---------|------|
| 招标公告发布信息 | 含"招标公告""项目编号""采购人""采购代理机构"的公告头 | "XX单位就XX项目进行公开招标" |
| 报名与缴费信息 | 含"报名费""招标文件售价""购买招标文件""投标保证金""开户银行""账户" | "招标文件售价500元，开户行XX" |
| 开标信息 | 含"开标时间""开标地点""投标截止" | "开标时间：2024年X月X日" |
| 联系方式 | 含"联系人""电话""邮箱""地址"且非技术联系人 | "项目联系人：张三，电话：XXX" |
| 无效政策引言 | 含"根据《XX法》""依据XX规定"且后续无具体执行条款的纯引言段落 | "根据《政府采购法》有关规定" |
| 评标委员会组成 | 含"评标委员会""评委""评审专家"组成规则 | "评标委员会由5人组成" |

### 1.2 必须保留的内容

| 保留类别 | 识别特征 |
|---------|---------|
| 投标人须知 | 含"投标人须""投标方应""须满足""应符合"的实质性要求 |
| 资格条件 | 含"资格要求""资质条件""准入条件" |
| 评分办法 | 含"评分""打分""得分""分值""评审因素" |
| 技术规格 | 含"技术参数""性能要求""规格""配置""功能要求" |
| 废标条款 | 含"废标""无效投标""否决""拒绝""不予受理" |
| 标书格式要求 | 含"装订""页码""盖章""签字""文件组成""格式" |

### 1.3 边界判定规则

| 情境 | 判定 | 处理方式 |
|------|------|---------|
| 同一段落既有废标条款又有评分条款 | 两类都保留 | 评分条款归入"技术评分细则区"，废标条款归入"废标否决条款区"，该段落在两区都出现，各自标注交叉索引 |
| 条款同时涉及技术参数和评分条件 | 按主旨归类 | 以条款的核心目的归类：主要目的为评分的归评分区，主要目的为技术要求的归技术区；归入一区后，在另一区添加交叉引用"详见X区第X条" |
| 条款表述模糊，难以判定类别 | 归入最相关区 | 优先归入"废标否决条款区"（宁可多识别废标风险），并在该条目标注【分类待确认】 |
| 含"保证金"字样但上下文为"保证金缴纳不足导致废标" | 保留 | 属于废标条款，归入"废标否决条款区" |

## 二、三分法拆分规则

### 2.1 第一区：技术评分细则区

**纳入标准**：所有与技术评分直接相关的条款。

**条目识别方法**：
- 含"技术部分""技术评审""技术评分""技术打分"章节下的所有条款；
- 含明确分值数字（如"X分""得X分""满分X分"）的技术类条款；
- 含"加分""扣分""得分条件"的技术类条款。

**排除标准**：
- 商务评分条款（含"商务评分""商务部分""企业资质评分""业绩评分""财务评分"）；
- 价格评分条款（含"价格分""报价评分""价格扣除"）。

**每条摘录格式**：
```
【条目编号】评分区-001
【原文条款】（逐字摘录招标原文，不改写）
【分值】X分
【得分条件】（从原文提炼：满足什么条件得满分，部分满足得多少分）
【佐证材料要求】（原文要求的证明文件、检测报告、证书等，如无则标注"原文未明确"）
【加/扣分规则】（原文明确的加分扣分条件，如无则标注"原文未明确"）
【来源索引】招标文件原文第X条/第X章第X节
```

### 2.2 第二区：技术需求规范区

**纳入标准**：所有技术参数、服务标准、交付要求。

**条目识别方法**：
- 含"技术参数""技术指标""性能要求""功能要求""配置要求"的条款；
- 含"服务期""运维期""质保期""工期""交付"的条款；
- 含"验收标准""交付物""实施范围"的条款；
- 含"软硬件""设备""系统""平台"配套要求的条款。

**排除标准**：
- 纯商务条款（付款方式、合同条款、违约责任中不涉及技术交付的部分）。

**每条摘录格式**：
```
【条目编号】技术区-001
【原文条款】（逐字摘录，不改写）
【参数/要求类别】设备参数 / 服务标准 / 工期要求 / 运维要求 / 交付标准 / 软硬件配套
【关键参数】（提取该条款中的量化指标：数量、规格、性能数值、时间节点）
【是否带★号或实质性要求标注】是/否（带★号或标注"实质性要求"的条款为不可偏离项）
【来源索引】招标文件原文第X条/第X章第X节
```

### 2.3 第三区：废标否决条款区

**纳入标准**：所有可能导致投标无效或被否决的条款。

**条目识别方法**：
- 含"废标""无效投标""否决投标""拒绝投标""不予受理"的条款；
- 含"实质性要求""实质性响应""★号条款"且明确写明不满足即废标的条款；
- 含"必须""应当""须"且上下文为资格条件或强制要求的条款（需标注【需人工确认是否属于废标条件】）；
- 资格审查条款（含"资格要求""准入条件"中不满足即否决的条款）。

**排除标准**：
- 商务/价格相关废标条款（如"报价超过预算废标""未缴纳保证金废标"）。

**每条摘录格式**：
```
【条目编号】废标区-001
【原文条款】（逐字摘录，不改写）
【废标类型】资格废标 / 资料缺失废标 / 格式错误废标 / 实质性不响应废标
【触发条件】（从原文提炼：缺少什么/不满足什么/做错什么会导致废标）
【是否带★号或"实质性要求"标注】是/否
【来源索引】招标文件原文第X条/第X章第X节
```
