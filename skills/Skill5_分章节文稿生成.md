# Skill5｜分章节文稿生成

【触发口令】调用技能5：分章节文稿生成

【执行前置】Skill4提示词+风控台账用户确认通过

【执行前必须读取】{项目}/02_目录与提示词/撰写提示词.md（仅此一份，严禁加载{项目}/02_目录与提示词/风控台账.md），以及本文件（投标编制规范.md中的角色定位、流转规则、权限红线）

【执行行为指引】
1. 向用户说明当前步骤："正在执行步骤5：分章节文稿生成，将按目录顺序逐章撰写技术方案，每章完成后等待您审阅"；
2. 按本Skill规则逐章撰写，每章产出后：
   - 向用户输出：文件已生成 `{项目}/03_章节初稿/第XX章_章节名.md`
   - 提示用户："请审阅后回复【本章定稿】继续下一章，或【本章驳回：原因】重写本章"
3. 收到【本章定稿】：归档本章，继续下一章节；
4. 收到【本章驳回：原因】：依据反馈重写本章；
5. 全部章节定稿后，自动进入步骤6全文合并，提醒用户："全部章节已定稿，正在执行步骤6：全文合并整编"。

---

## 一、撰稿执行流程

### 1.1 章节撰写顺序

- 严格按{项目}/02_目录与提示词/标书目录.md的顺序自上而下撰写；
- 单次只输出一个完整章节，禁止一次性输出多个章节；
- 上一章用户定稿后，再开始下一章。

### 1.2 单章节撰写步骤

1. 从{项目}/02_目录与提示词/撰写提示词.md读取当前章节的提示词；
2. 从{项目}/01_解析与基准/招标文件解析.md读取该章节【招标原文锚点】对应的原文条款（仅用于对标参考，不逐字抄入正文）；
3. 按提示词的【段落结构】建议组织段落；
4. 按提示词的【内容范畴】逐一展开写作模块；
5. 按提示词的【内容边界】控制写作范围，不越界；
6. 对照提示词的【图表要求】，在对应位置插入图表说明占位（见下文3.3节）；
7. 输出完整章节，提示用户审阅。

## 二、正文撰写硬性约束

### 2.1 语言风格

| 要求 | 具体规则 | 违规示例 |
|------|---------|---------|
| 公文语体 | 使用正式书面语，句式完整，主谓宾齐全 | "我们搞了一个方案"→"本项目设计了如下方案" |
| 禁用口语 | 不得使用网络用语、口语化表达、感叹句 | "超级强大""非常厉害""好！" |
| 禁用夸张修饰 | 不得使用"业界领先""国际一流""独一无二"等无法验证的修饰 | "采用业界领先的技术"→"采用XX技术" |
| 客观陈述 | 以陈述句为主，避免主观评价 | "我们的方案是最佳的"→"本方案满足XX要求" |
| 被动语态适度 | 技术描述可使用"本系统采用""本项目部署"等客观表述 | — |

### 2.2 内容对标规则

| 规则 | 具体要求 |
|------|---------|
| 所有技术描述对标招标原文 | 提到的技术参数、服务标准、交付要求必须能在招标文件中找到依据 |
| 禁止自定义参数 | 不得出现招标文件未提及的产品型号、版本号、性能指标 |
| 禁止自创服务年限 | 服务期、运维期、质保期必须与招标文件一致，不得自行延长或缩短 |
| 禁止编造配套方案 | 不得自行添加招标未要求的子系统、模块、功能 |

### 2.3 占位符使用规则

无法确认的信息，统一使用占位符标注，AI不得自行编造：

| 占位类型 | 格式 | 示例 |
|---------|------|------|
| 企业资质 | 【【人工补充：XXX资质证书信息】】 | 【【人工补充：ISO27001认证证书编号及有效期】】 |
| 项目案例 | 【【人工补充：XXX案例信息】】 | 【【人工补充：近3年同类项目案例名称及合同金额】】 |
| 产品参数 | 【【人工补充：XXX产品参数】】 | 【【人工补充：核心交换机具体型号及性能参数】】 |
| 人员信息 | 【【人工补充：XXX人员信息】】 | 【【人工补充：项目经理姓名、资质、履历】】 |
| 企业数据 | 【【人工补充：XXX数据】】 | 【【人工补充：公司注册资金、员工人数】】 |

**占位符放置规则**：
- 占位符放在需要补充内容的位置，前后保持语句通顺；
- 示例："本项目团队配备项目经理1名，【【人工补充：项目经理姓名、资质、履历】】，负责项目整体管理与协调。"

## 三、段落与结构规范

### 3.1 段落组织模式

根据提示词中的【段落结构】建议，采用以下标准模式：

**"总-分"式**（适用于概述类章节）：
```
第1段：总述/概述（本节核心结论或整体情况）
第2-N段：分点展开（各要点详细阐述）
末段：小结（呼应总述，可省略）
```

**"问题-方案"式**（适用于需求响应类章节）：
```
第1段：问题/需求描述（招标文件提出的需求）
第2-N段：解决方案（逐条回应需求）
末段：方案优势/效果（简述方案如何满足需求）
```

**"递进"式**（适用于设计类章节）：
```
第1段：设计原则/目标
第2段：总体设计思路
第3-N段：逐层深入展开
末段：设计总结
```

**"并列"式**（适用于多点罗列类章节）：
```
第1段：引言（概述本节涵盖的要点）
第2段：要点一
第3段：要点二
……
```

### 3.2 段落长度控制

| 位置 | 建议字数 |
|------|---------|
| 概述/总述段 | 100-200字 |
| 分述段 | 150-300字 |
| 小结段 | 50-100字（可省略） |

### 3.3 图表插入规则

| 图表类型 | 插入格式 | 示例 |
|---------|---------|------|
| 架构图 | 【图：XXX架构图】 | 【图：总体网络架构图】 |
| 流程图 | 【图：XXX流程图】 | 【图：项目实施流程图】 |
| 参数表 | 【表：XXX参数对比表】 | 【表：核心设备性能参数表】 |
| 清单表 | 【表：XXX清单】 | 【表：项目交付物清单】 |

**图表说明规则**：
- 图表前必须有引导文字，如"本项目总体架构如下图所示："；
- 图表后必须有解读文字，如"如上图所示，系统分为……三层"；
- 图表占位符仅标注类型和名称，不绘制实际图表内容（图表由人工后续补充或用工具生成）。

## 四、章节间交叉引用规则

### 4.1 引用格式

- 引用其他章节：使用"详见第X章XX节"格式；
- 引用本章节其他段落：使用"如前文所述"或"见下文XX部分"；
- 示例："核心交换机具体选型方案详见第3章3.2节。"

### 4.2 引用一致性

- 前文提到的方案，后文必须有对应展开；
- 后文展开的内容，前文必须有过提及或铺垫；
- 引用的章节编号必须与{项目}/02_目录与提示词/标书目录.md一致，不可编造章节号。

## 五、输出文件结构

每章节输出独立文件至 `{项目}/03_章节初稿/` 目录下，命名格式：`第XX章_章节名.md`

```markdown
# 第X章 章节名称

## X.1 二级标题

### X.1.1 三级标题

（正文内容）

本项目采用【【人工补充：核心交换机具体型号及性能参数】】，满足招标文件对核心交换机的性能要求。核心交换机具体选型方案详见第3章3.2节。

【图：总体网络架构图】

如上图所示，网络架构分为核心层、汇聚层、接入层三层设计……

### X.1.2 三级标题

（正文内容）
……

## X.2 二级标题
……
```

## 六、执行后操作

1. 每章节输出后，向用户提示："文件 {项目}/03_章节初稿/第XX章_章节名.md 已生成，请审阅后回复【本章定稿】或【本章驳回：原因】"；
2. 用户回复【本章定稿】：归档本章内容，开始下一章节撰写；
3. 用户回复【本章驳回：原因】：依据反馈重写本章，不得沿用旧版内容；
4. 全部章节定稿后，自动进入步骤6全文合并。

---

【禁止行为】
- 跳章节撰写（必须按目录顺序）
- 一次性输出多个章节
- 加载{项目}/02_目录与提示词/风控台账.md文件
- 自行填充企业信息与数据（必须使用占位符）
- 脱离招标条款自创方案、自定义参数
- 在正文中提及评分分值、废标条款、扣分规则
- 编造产品型号、版本号、性能指标
