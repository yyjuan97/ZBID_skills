# Skill8｜项目全套资料归档

【触发口令】调用技能8：项目全套资料归档

【执行前置】排版定稿，全流程9工序完结

【执行前必须读取】全部前置产出文件，以及本文件（投标编制规范.md中的角色定位、流转规则、权限红线）

【执行行为指引】
1. 向用户说明当前步骤："正在执行步骤9：项目全套资料归档，将校验全部文件完整性并生成归档清单"；
2. 按本Skill规则执行校验和归档，产出文件至项目文件夹；
3. 执行完毕后向用户输出以下信息：
   - 文件已生成：`{项目}/06_归档/归档清单.md`
   - 校验结果：完整X项/缺失X项，章节数量一致/不一致，质检闭环/未闭环
   - 提示用户："请审阅归档清单后确认项目闭环"
4. 用户确认后，项目正式闭环，提示用户："项目全流程9工序已完结"。

| 章节数量一致 | {项目}/02_目录与提示词/标书目录.md中的章节数 vs 实际{项目}/03_章节初稿/下文件数 | 在归档清单中标注差异 |
| 质检问题已闭环 | {项目}/04_合并与质检/质检报告.md中的问题数量 vs 复检报告.md中已通过数量 | 未闭环问题在归档清单中标注 |

- 存在缺失文件：在归档清单中标注【缺失】，不阻断归档流程；
- 存在不一致：在归档清单中标注差异说明，由用户决定是否补充；
- 全部通过：直接输出归档清单。
