## 一、提示词的层级分类与写作指引差异

目录可能包含多层嵌套，不同层级的章节写作要求不同，提示词必须区分：

| 层级类型 | 定位 | 写作要求 | 提示词侧重 |
|---------|------|---------|-----------|
| 概述型章节 | 评分项/技术领域的总体介绍 | 概括性描述，引出下文详细设计 | 内容范畴+段落结构+承上启下 |
| 详细设计型章节 | 技术子领域的展开说明 | 既有总体说明又有分项设计 | 内容范畴+段落结构+图表要求 |
| 逐条响应型章节 | 对应技术响应表单条技术要求 | **逐条响应，必须明确回答招标参数** | 技术要求编号+响应方式+响应内容 |

**关键区别**：逐条响应型章节是提示词的最低粒度，每条对应技术响应表中的一行（T-XXX），提示词必须明确该条技术要求的响应方式。

## 二、正文撰写提示词结构模板

### 2.1 概述型/详细设计型章节的提示词

适用于：目录中第1-3层的章节（评分项、评分点、技术子领域）。

```
【目录编号】3.2
【章节标题】核心层网络详细设计
【章节类型】详细设计型
【内容范畴】①概述核心层网络设计目标与原则；②引出下文各核心设备选型与配置
【内容边界】本节仅概述核心层设计，具体设备选型详见3.2.1-3.2.3；不涉及汇聚层/接入层
【建议字数】300-500字
【段落结构】"总-分"式：第1段概述核心层设计目标，第2段说明核心层在网络架构中的定位，第3段引出下文设备选型
【图表要求】建议配1张核心层网络拓扑图
【下级章节】3.2.1、3.2.2、3.2.3
【招标原文锚点】技术区-003、技术区-004
```

### 2.2 逐条响应型章节的提示词（⚠️核心新增）

适用于：目录中对应技术响应表单行技术要求的最底层章节。

```
【目录编号】3.2.1
【章节标题】核心交换机选型与配置
【章节类型】逐条响应型
【技术要求编号】T-001
【招标技术原文】核心交换机：背板带宽≥2Tbps
【招标参数要求】背板带宽≥2Tbps
【响应方式】参数响应
【响应内容】①说明核心交换机选型依据与选型方案；②逐项响应背板带宽参数，说明本方案如何满足；③描述核心交换机在网络中的部署位置与角色
【内容边界】仅响应T-001核心交换机背板带宽要求，其他核心交换机参数详见3.2.2、3.2.3
【建议字数】400-600字
【段落结构】"问题-方案"式：第1段说明招标对核心交换机背板带宽的要求，第2段描述本方案选型及参数响应，第3段说明部署方案
【图表要求】无（或：建议配1张核心交换机部署位置示意图）
```

### 2.3 各字段填写规则

**【章节类型】**：
- 根据目录层级和技术响应表对应关系判断；
- 有对应技术响应表编号（T-XXX）的 → "逐条响应型"；
- 无直接对应技术响应表编号，属于概述/分类/引出下文的 → "概述型"或"详细设计型"。

**【技术要求编号】**（仅逐条响应型填写）：
- 从{项目}/01_解析与基准/技术响应表.md中提取该章节对应的T编号；
- 一条提示词只对应一个T编号，不可多对一。

**【招标技术原文】**（仅逐条响应型填写）：
- 从{项目}/01_解析与基准/技术响应表.md中"招标技术原文"列逐字摘录。

**【招标参数要求】**（仅逐条响应型填写）：
- 从{项目}/01_解析与基准/技术响应表.md中"招标参数要求"列摘录。

**【响应方式】**（仅逐条响应型填写）：

| 技术要求类型 | 响应方式 | 提示词中写法 |
|------------|---------|-------------|
| 量化参数（如背板带宽≥2Tbps） | 参数响应 | "逐项响应XX参数，说明本方案如何满足" |
| 定性要求（如提供7×24小时运维） | 方案响应 | "描述运维服务方案，说明如何满足7×24小时要求" |
| 设备/系统选型 | 选型响应 | "说明XX设备/系统的选型依据、选型方案、参数对比" |
| 功能要求（如支持XX功能） | 功能响应 | "描述XX功能的实现方案，说明技术实现路径" |
| 实施类要求（如工期XX天） | 实施响应 | "描述实施计划，说明如何在XX天内完成" |

**【响应内容】**（仅逐条响应型填写）：
- 将技术要求转化为2-4个写作模块；
- 每个模块用一句话描述；
- 示例："①说明核心交换机选型依据与方案；②逐项响应背板带宽参数；③描述部署位置与角色"。

**【内容范畴】**（仅概述型/详细设计型填写）：
- 从{项目}/02_目录与提示词/标书目录.md确定章节定位；
- 从{项目}/01_解析与基准/招标文件解析.md提取该章节对应的技术需求/评分条目；
- 将需求转化为2-5个写作模块。

**【内容边界】**：
- 明确本节不涉及的内容，防止与同级或下级章节重复；
- 概述型：不涉及具体参数响应（详见下级章节）；
- 逐条响应型：仅响应本条T编号，其他T编号详见对应章节。

**【建议字数】**：
- 逐条响应型章节：300-600字（单条技术要求）；
- 详细设计型章节：400-800字；
- 概述型章节：200-500字；
- 无直接评分的支撑章节：200-400字。

**【段落结构】**：
- 从以下模式中选择或组合：
  - "总-分"式：先概述再分点展开；
  - "问题-方案"式：先描述需求再给方案；
  - "递进"式：按逻辑层次逐步深入；
  - "并列"式：多个独立要点平行展开。

**【图表要求】**：
- 架构类：建议配"架构拓扑图"；
- 流程类：建议配"流程图"；
- 参数对比类：建议配"参数对比表"；
- 设备部署类：建议配"部署位置示意图"；
- 无需图表时填写"无"。

**【下级章节】**（仅概述型/详细设计型填写）：
- 列出本章节下的直接子章节编号，便于撰稿时承上启下。

**【招标原文锚点】**：
- 列出该章节对应的{项目}/01_解析与基准/招标文件解析.md中的条目编号；
- 逐条响应型：锚点与技术要求编号对应。

### 2.4 提示词编写禁止

| 禁止内容 | 原因 |
|---------|------|
| 评分分值 | 属于风控信息 |
| 扣分规则 | 同上 |
| 废标风险提示 | 同上 |
| 技术参数硬性约束值 | 如"必须≥500TB"，提示词中只写"阐述存储容量方案" |
| "必须""不得"等强制性表述 | 提示词是写作指引，不是约束条款 |

## 三、风控台账结构模板

### 3.1 每条台账固定字段

每条台账与提示词按目录编号一一绑定，按以下固定结构编写：

```
【目录编号】3.2.1（与提示词一致）
【章节标题】核心交换机选型与配置（与提示词一致）
【章节类型】逐条响应型
【技术要求编号】T-001
【对应评分条目】
  - P-005：网络架构及设备选型
【招标技术硬性约束】
  - T-001：背板带宽≥2Tbps
【废标风险提示】
  - 本章无废标风险
```

### 3.2 各字段填写规则

**【章节类型】**、**【技术要求编号】**：
- 与提示词保持一致。

**【对应评分条目】**：
- 从{项目}/01_解析与基准/评分对标表.md提取该章节对应的所有评分条目；
- 格式：条目编号 + 评分项名称；
- 该章节无直接评分条目的，填写"本章无直接评分条目"。

**【招标技术硬性约束】**：
- 从{项目}/01_解析与基准/技术响应表.md提取该章节涉及的所有量化参数要求；
- 格式：技术要求编号 + 参数要求原文；
- 仅提取带"≥""≤""不少于""不超过"等硬性约束的参数；
- 该章节无硬性约束的，填写"本章无硬性技术约束"。

**【废标风险提示】**：
- 从{项目}/01_解析与基准/废标核查表.md提取该章节涉及的所有废标条款；
- 重点标注★号条款和实质性要求；
- 该章节无废标风险的，填写"本章无废标风险"。

### 3.3 台账使用规则

- 台账仅在Skill6（合规预检）时调取；
- Skill5（分章节撰写）执行时严禁读取台账文件；
- 台账内容严禁以任何形式出现在标书正文中。

## 四、两份文档的编号绑定规则

| 规则 | 说明 |
|------|------|
| 编号一致 | 提示词与台账的【目录编号】必须完全一致 |
| 逐条对应 | 目录中每个编号在两份文档中各有一条记录，不可遗漏 |
| 层级一致 | 提示词与台账的【章节类型】【技术要求编号】必须一致 |
| 顺序一致 | 两份文档按目录编号顺序排列 |
| 独立存档 | 两份文档物理分离，不可合并 |

## 六、执行后操作

1. 输出两个文件后，向用户提示："文件 {项目}/02_目录与提示词/撰写提示词.md、风控台账.md 已生成，请审阅后回复【确认通过】或【驳回重做：XX问题】"；
2. 重点提示用户检查：提示词中是否混入了评分分值/废标条款（如有则必须驳回）；
3. 重点提示用户检查：逐条响应型章节是否与技术响应表每行一一对应（如有遗漏则驳回）；
4. 用户确认通过后，数据带入Skill5；
5. 用户驳回时，依据驳回说明重新生成，不得沿用旧版内容。

【禁止行为】
- 将风控台账内容混入撰写提示词
- 在提示词中提及评分分值或废标条款
- 统一套用无项目针对性通用模板
- 提示词与台账编号不一致
- 遗漏目录中任何一个编号
- 逐条响应型章节缺少技术要求编号（T-XXX）
- 多条技术要求合并为一条提示词（必须一对一）
