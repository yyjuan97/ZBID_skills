## 它解决什么问题

编写投标技术方案时，你是否遇到这些痛点：

- **评审找不到对应得分点** — 方案目录和评分表对不上，评审翻遍标书找不到响应位置
- **技术要求遗漏** — 招标文件参数多、表格密，写完才发现漏了好几条
- **风控内容误入正文** — 评分分值、废标条款写进标书，反而成为扣分把柄
- **反复修改格式** — 每次驳回都是重写，缺少结构化流程管控

ZBID通过**9步固定流程 + 9个独立Skill技能文件**，让AI按规则逐步执行，每步产出文件供你审阅确认，确保技术方案完整响应每条要求，且风控内容与正文物理隔离。

---

## 核心理念

**目录结构 = 评分表镜像，每条技术要求逐条响应**

- 章节对齐评分项，评审翻目录就能找到对应得分点
- 技术响应表每行（T-001、T-002……）= 目录中一个子章节 = 提示词一条 = 正文一个响应小节
- 撰写提示词与风控台账物理分离，风控内容绝不进入正文

---

## 9步流程

```
步骤1 招标文件解析         → 步骤2 三份基准表单         → 步骤3 评分结构目录
步骤4 提示词+风控台账       → 步骤5 逐章撰写（循环）     → 步骤6 全文合并
步骤7 合规预检             → 步骤8 标准化排版           → 步骤9 归档闭环
```

每步产出独立文件，你审阅确认后才进入下一步，AI不可跳步、不可自行推进。

## 快速开始

### 1. 准备文件

```
你的工作目录/
├── 投标编制规范.md              ← 规范主文件
└── skills/                      ← 技能文件目录
    ├── Skill1_招标文件智能解析拆分.md
    ├── Skill2_生成三份基准文档.md
    ├── Skill3_标书分级目录搭建.md
    ├── Skill4_批量生成撰写提示词库与风控台账.md
    ├── Skill5_分章节文稿生成.md
    ├── Skill6_三重合规AI预检.md
    ├── Skill7_标准化排版.md
    ├── Skill8_项目全套资料归档.md
    └── Skill_补充招标文件追加解析.md
```

### 2. 启动

在AI对话窗口（Claude Code / ChatGPT 等支持文件读取的工具）中发送：

```
请阅读 投标编制规范.md，理解你的角色定位、作业流程和权限红线，然后告诉我你已准备好。
```

### 3. 开始编制

```
调用技能1：招标文件智能解析拆分，项目名称：XX项目，招标文件路径：./招标文件/XX项目招标文件.pdf
```

AI会自动创建项目文件夹、执行解析、产出文件、提示你审阅。

## 项目文件夹结构

每个项目自动创建独立文件夹，按流程阶段分类存放：

```
projects/{项目名称}/
├── 00_招标文件/              ← 原始招标文件、补充文件
├── 01_解析与基准/            ← 解析结果 + 三张基准表
├── 02_目录与提示词/          ← 标书目录 + 撰写提示词 + 风控台账
├── 03_章节初稿/              ← 每章独立文件
├── 04_合并与质检/            ← 合并初稿 + 质检/复检报告
├── 05_排版终稿/              ← 排版定稿
└── 06_归档/                  ← 归档清单
```

## 各步骤详解

### 步骤1：招标文件解析

```
调用技能1：招标文件智能解析拆分，项目名称：XX项目，招标文件路径：XXX
```

AI清洗无关内容（公告、缴费、联系方式），按三分法拆分：
- **技术评分细则区** — 逐条提取技术评分项、分值、得分条件
- **技术需求规范区** — 逐条提取技术参数、服务标准、交付要求
- **废标否决条款区** — 逐条提取废标条件（过滤商务/价格废标项）

产出：`01_解析与基准/招标文件解析.md`

### 步骤2：三份基准表单

```
调用技能2：生成三份基准文档
```

| 表单 | 内容 | AI填充范围 |
|------|------|-----------|
| 评分对标表 | 技术评分条目逐条列出 | 前3列（AI），后3列留白（人工） |
| 技术响应表 | 每条技术要求独立一行，带T编号、★号标注 | 前3列（AI），后3列留白（人工） |
| 废标核查表 | 技术相关废标条款逐条列出 | 前2列（AI），后3列留白（人工） |

产出：`01_解析与基准/` 下三个独立文件

### 步骤3：评分结构目录

```
调用技能3：标书分级目录搭建
```

目录=评分表镜像，层级不限（4-5层正常）：
- 章 → 评分大类
- 节 → 评分项（名称体现评分项关键词）
- 小节 → 技术子领域
- 条 → 技术响应表每行（T-001、T-002……）必须有独立标题

产出：`02_目录与提示词/标书目录.md`

### 步骤4：撰写提示词 + 风控台账

```
调用技能4：批量生成撰写提示词库+风控台账
```

两份文档物理分离：

| 文档 | 内容 | 用途 |
|------|------|------|
| 撰写提示词 | 概述型 / 详细设计型 / 逐条响应型（绑定T编号） | 供步骤5撰稿使用，**不含**评分分值和废标条款 |
| 风控台账 | 评分条目+分值 + 技术硬性约束 + 废标风险提示 | 仅供步骤7质检使用，**严禁**进入正文 |

产出：`02_目录与提示词/` 下两个独立文件

### 步骤5：逐章撰写

```
调用技能5：分章节文稿生成
```

- 逐章输出，每章一个文件
- 仅读取撰写提示词，**严禁读取风控台账**
- 无法确认的信息用【【人工补充：XXX】】占位
- 每章审阅后回复：`本章定稿` → 下一章 / `本章驳回：原因` → 重写

产出：`03_章节初稿/第XX章_章节名.md`

### 步骤6：全文合并

自动执行，无需手动触发。按目录顺序串联定稿章节，统一编号，处理衔接。

产出：`04_合并与质检/合并初稿.md`

### 步骤7：合规预检

```
调用技能6：三重合规AI预检
```

四维核查，风险分级：

| 维度 | 风险等级 |
|------|---------|
| 废标缺项 | 高危 |
| 评分漏项 | 中风险 |
| 技术偏离 | 中风险 |
| 占位符残留 | 高危 |

用户整改后回复`整改完成`触发复检，仅复检整改相关项。

产出：`04_合并与质检/质检报告.md` + `复检报告.md`（如有）

### 步骤8：排版

```
调用技能7：标准化排版
```

按招标格式要求标注标题字体、行距、缩进、签章位置等排版规范。

产出：`05_排版终稿/排版终稿.md`

### 步骤9：归档

```
调用技能8：项目全套资料归档
```

校验全部文件完整性，生成归档清单，项目闭环。

产出：`06_归档/归档清单.md`

## 数据链路

技术响应表是整条链路的锚点，每行严格一一对应：

```
T-001  ──→  目录子章节  ──→  提示词(逐条响应型)  ──→  正文响应小节
T-002  ──→  目录子章节  ──→  提示词(逐条响应型)  ──→  正文响应小节
T-003  ──→  目录子章节  ──→  提示词(逐条响应型)  ──→  正文响应小节
```

链路任何一环断裂 = 该技术要求未被响应。步骤7质检会反向核查，但最好在步骤2-4就把关。

## 权限红线

1. AI无权修改基准表格固定列结构
2. AI无权标注正偏离/负偏离
3. AI不得编造资质、案例、参数
4. AI不可跳过用户确认环节
5. 风控台账严禁进入正文
6. 撰写时严禁加载风控台账文件
